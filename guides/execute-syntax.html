🎯 新executeコマンド完全解説

統合版で新しくなった「/execute」コマンドの書き方と、11個の「サブコマンド」の使い方をぜんぶ解説するよ！

1

新execute（エグゼキュート）ってなに？

/execute は、「だれが」「どこで」「どんなときに」コマンドを実行するかを細かく決められる、マイクラで一番重要といってもいい超強力なコマンドだよ！

統合版（Switchやスマホ）のバージョン1.19.50から、それまでの古い書き方（旧構文）から、より高機能な「新構文」にアップデートされました。

新executeのメリット
昔の書き方に比べて、英語のパーツ（サブコマンド）をパズルのように何個でもつなげて、複雑な条件をかんたんに書くことができるようになったんだ！
2

基本の書き方のルール

新executeは、いくつかのパーツ（サブコマンド）を繋げて書いて、最後に実行したいコマンドを run のあとに書きます。

/execute [サブコマンド1] [サブコマンド2] ... run [動かしたいコマンド]

たとえば、「すべてのゾンビの場所（at）で、雷を落とす（summon）」場合はこのように書きます。

3

11個のサブコマンド一覧

統合版で使える11個のサブコマンドの役割をまとめたよ！

サブコマンド 役割（何をするもの？）
as 「だれ」が実行するか（実行する人・エンティティ）を変える。
at 「どこで」実行するか（場所や向き、ディメンション）を指定エンティティと同じにする。
positioned 実行する「座標（座標のみ）」を別の場所や指定エンティティの位置に変える。
rotated 実行時の「向き（回転角度）」を特定の角度や指定エンティティの向きに変える。
facing 実行時の向きを指定した「座標」や「エンティティ」の方向に向けさせる。
anchored ローカル座標（^ ^ ^）を使うときの基準位置を「目（eyes）」または「足元（feet）」にする。
align 実行座標の小数点を丸めて、「ブロックの角」に位置をぴったり揃える。
in 実行する「ディメンション（世界）」（オーバーワールド、ネザー、エンド）を変える。
if もし〜なら（指定した条件が合っているとき）コマンドを実行する。
unless もし〜でないなら（指定した条件が間違っているとき）コマンドを実行する。
run サブコマンドのあとに、「実際に実行したいコマンド」を指定する（必須の締め言葉）。
4

各サブコマンドのくわしい使い方

1. as（〜として実行）

コマンドの実行者（「だれが言ったか」の主語）を変更するよ！対象モブに @s （自分自身）で自分を対象にさせたいときなどに使います。

注意： as は実行する「人」を変えるだけなので、実行される「場所」は変わりません！（コマンドブロックの位置や、元のプレイヤーの位置のまま）。場所も変えたい場合は、このあ後の at も一緒に使いましょう！

2. at（〜の場所で実行）

コマンドを実行する「座標（場所）」「向き（回転角度）」「ディメンション」を、指定したモブと同じものに変更します。場所だけを変えたいときは at が一番よく使われます。

3. as と at の組み合わせ（超定番！）

「ゾンビになって（as）、そのゾンビのいる場所で（at）実行する」というように組み合わせて使うことで、エンティティを基準にした仕掛けを完ぺきに作れます。

4. positioned（位置だけを変更）

実行する「座標（場所）」だけを変更します。at と違って、実行する向き（回転角度）やディメンションは変更しません。

  • positioned ~ ~5 ~: 今の場所から5マス上の座標に変更する。
  • positioned as @p: プレイヤーと同じ座標に変更する（向きは変わらない）。

5. rotated（向きを変更）

実行時の「向き（回転角度）」を変更します。プレイヤーやモブの向きに合わせて別の処理をしたいときに使います。

  • rotated 0 0: 北を向いている（水平0度、垂直0度）状態にします。
  • rotated as @p: プレイヤーが見ている向きと同じにします。

6. facing（〜の方向を向く）

実行位置から、指定した座標やエンティティの方を向かせた状態にします。ローカル座標 ^ を使って「指定したモブの方向へ何かを飛ばす・動かす」というときに大活躍します！

  • facing 0 60 0: 座標「0, 60, 0」の方向に向かせます。
  • facing entity @p eyes: プレイヤーの「目（eyes）」の方向に向かせます（足元を向かせる場合は `feet`）。

7. anchored（目の高さ基準）

実行座標の基準（アンカー）を「足元（feet）」にするか「目（eyes）」にするかを設定します。ローカル座標 ^ ^ ^ を使って「見ている方向にビームやブロックを出す」というときに、目線の高さから出すために使います。

8. align（グリッドに揃える）

実行座標の小数点を切り捨てて、ブロックの境界にぴったり揃えます（ブロックの角や中心に位置をあわせるのに便利です）。指定した軸（x、y、z）で揃えられます。

9. in（世界を変更）

実行するディメンション（世界）を変更します。異なるディメンションにいるプレイヤーやモブを検知・操作したいときに使います。

  • in overworld: 通常世界（オーバーワールド）
  • in nether: 暗黒界（ネザー）
  • in the_end: 果ての世界（ジ・エンド）
5

条件を決める if と unless

新executeでは、if （もし〜なら）と unless （もし〜でなければ）を使って、コマンドの実行条件を細かく決められます。

※統合版には Java版にある if data や if predicate などは存在しないため、以下の4つの条件を使います。

① block（特定のブロックがあるか）

指定した座標に、特定のブロックがあるかどうかを調べます。

② blocks（エリアのブロックが一致するか）

指定した「開始位置から終了位置までのエリア」のブロックが、「目的地エリア」のブロックと同じかどうかを調べます。

  • if blocks all: すべて完全に一致するか。
  • if blocks ... masked: 空気以外のブロックが一致するか（空気部分は無視）。

③ entity（特定のモブやプレイヤーがいるか）

指定したターゲット（モブやプレイヤー）が存在するかどうかを調べます。一番よく使われます。

④ score（スコアボードの点数を比べる）

スコアボードの点数を比べたり、範囲内に入っているかを調べます。

  • スコアの比較演算子: <, <=, =, >, >=
  • matches [範囲]: スコアが範囲内に入っているか（例：`1..5` は 1〜5点、`..10` は 10点以下、`5..` は 5点以上）。
6

複数のパーツを繋げてみよう！（実践例）

これらのパーツを繋げて、複雑な仕掛けを作ってみよう！

例①：「金ブロックの上にいる時だけ、スピードが早くなる」

as, at, if block をつなぎます。

例②：「ゾンビがプレイヤーから15マス以内に近づいた時、ゾンビが燃え出す」

as, at, if entity をつなぎます。
